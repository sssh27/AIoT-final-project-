## 核心設計對話摘要

### 1. 系統架構決策 (System Architecture)
**用戶提問**：如何設計一個可擴展且具備原生感的校園 App？
**AI 建議**：
- 推薦採用 **MVVM (Model-View-ViewModel)** 模式搭配 **SwiftUI**。
- **理由**：SwiftUI 的宣告式 UI 能自動驅動介面更新，適合處理動態課表與位置提醒；MVVM 能將邏輯（如排課計算法）與 UI 分離，提高程式碼可維護性。

### 2. 衝堂偵測演算法討論 (Algorithm Design)
**用戶提問**：排課系統如何快速判斷時間重疊，並找出空的通識課？
**AI 建議**：
- **衝堂偵測**：使用 `Set.isDisjoint` 進行 $O(n^2)$ 遍歷。雖然時間複雜度看來較高，但因學生課表數量小（$n < 15$），在行動裝置端能達到毫秒級反應。
- **通識推薦**：利用 **Bitmask Matching** 或 `isSubset(of:)` 操作，將課程需求節次與使用者的「空堂」進行二進位或集合比對，確保推薦的精準度。

### 3. 用戶體驗與效能優化 (UX & Performance)
**討論焦點**：
- **資料管線**：實作 `CourseDataStore` 作為 **單例快取 (Singleton Cache)**，避免重複讀取逾千筆 CSV 資料，大幅節省電力與記憶體。
- **佈署策略**：採取 **Local-First (本地優先)** 儲存機制，確保學生隱私資料不經過外部伺服器，達到零攻擊面的安全性目標。
